# 新大フリマ UI/UX ブリーフ

> 画面一覧は engineering の `docs/mvp-architecture.md`「画面一覧」を基準とする。
> 本ブリーフでは実装着手前の **トーン・色・ワイヤー方針** までを定義する。

## 目的

新潟大学生が「安心して・直感的に」中古の教科書/ゲームを売買できる体験。
スマホ片手に数タップで出品・問い合わせができることを最優先する。

## ターゲット

- 新潟大学の学部生・院生（スマホ利用が中心、メルカリ等のフリマUIに慣れている）
- 利用シーン: 講義の合間、移動中、学期始め/終わりの教科書整理

## トーン・雰囲気

- **安心・誠実・学生らしい軽やかさ**。フォーマルすぎず、チャラすぎず。
- 「非公式・個人運営」を隠さず、誠実さで信頼を得る（免責は小さくとも常時表示）。
- 写真（商品画像）が主役。UIは脇役として情報を邪魔しない。

## カラー（実装済みの方針を踏襲）

| 用途 | 値 | メモ |
|------|----|------|
| ブランド（緑） | `--brand` `#0b7a4b` | 新潟大学らしい緑系。CTA・アクティブ・送信ボタン |
| ブランド濃 | `--brand-dark` `#075c38` | hover |
| 背景 | `#f7f7f8` | 画面全体。カードの白を引き立てる |
| 前景（文字） | `#1a1a1a` | 本文 |
| 注意・免責 | amber系(`bg-amber-50`/`text-amber-900`) | フッター免責 |
| 安全バナー | red系(`bg-red-50`/`text-red-800`) | チャット内ガイドライン |
| 売却済み | 黒70%バッジ＋白50%オーバーレイ | 一覧/詳細 |

- フォント: Hiragino / Noto Sans JP 系のゴシック（既定）。視認性重視。
- 角丸: カード `rounded-xl`、ボタン/入力 `rounded-xl`〜`full`。やわらかい印象。

## 主要画面のワイヤー方針

### ① ホーム `/`（最重要）
```
[ヘッダー: 新ロゴ | メッセージ | 出品(緑) | アバター]
[🔍 検索バー（全幅）                         ]
[ (すべて)(教科書)(ゲーム) ← 横スクロールchip ]
[商品グリッド 2列(SP)/3列(PC)               ]
  ┌────────┐┌────────┐
  │ 正方形画像 ││ 正方形画像 │   ← 画像が主役
  │ タイトル2行 ││          │
  │ ¥価格(太) ││ 無料     │
  └────────┘└────────┘
[フッター: 免責(amber) / ©]
```
- 方針: 画像ファースト、価格は太字。売却済みは淡くオーバーレイ。空状態は「最初の出品をする」CTA。

### ② 出品 `/items/new`
- 縦1カラム。**画像→商品名→カテゴリ→価格→説明→出品ボタン(緑/全幅)** の順。
- 画像枠は圧縮の段階表示（「画像を圧縮中…」→「アップロード中…」）とサムネ、完了時に「WebP/◯◯KB」を緑で表示。
- 「無料で譲る」チェックで価格入力を隠す。価格下に対面受け渡しの注意。

### ③ 商品詳細 `/items/[id]`
- 上: 画像（正方形〜16:9）。下: タイトル→価格(大)→カテゴリchip→説明→出品者→CTA。
- 他人の出品: 「出品者に問い合わせる」(緑/全幅)。自分の出品: ステータス切替(販売中/取引完了)＋削除。
- 末尾に「学内の安全な場所で・支払いは対面」の注意書き。

### ④ チャット `/chat/[id]`
- 最上部に**赤系の安全ガイドラインバナーを常時固定**（第一学生食堂前・図書館ラウンジ等で日中取引、密室禁止）。
- 右=自分(緑吹き出し)/左=相手(白吹き出し)。最下部に入力欄＋送信(緑・丸)。新着は自動スクロール。
- ヘッダーに相手名と対象商品サムネ（タップで商品詳細へ）。

### ⑤ メッセージ一覧 `/chat`・⑥ マイページ `/profile`
- 一覧: 行＝[商品サムネ][相手名/商品名/最新メッセージ][時刻]。LINEのトーク一覧に近い。
- マイページ: アバター＋名前＋メール、ログアウト、自分の出品グリッド。

## 要件 / 制約

- **モバイルファースト**（375px基準）、`max-w-3xl` 中央寄せでPCも破綻させない。
- アクセシビリティ: コントラスト確保、タップ領域44px以上、画像に alt。
- 免責表記は全画面の導線上（フッター/ログイン）に必ず表示。

## Figma 方針

- ページ構成: `00 Foundations`（色・タイポ・余白・コンポーネント）/ `01 Mobile`（①〜⑦のSP）/ `02 Desktop`（主要画面のみ）。
- 既存実装のTailwindトークンとFigmaのスタイルを一致させる（色は上表の値をそのまま登録）。
- コンポーネント化: ItemCard / ヘッダー / フッター / チャット吹き出し / 安全バナー / ボタン(primary緑)。
